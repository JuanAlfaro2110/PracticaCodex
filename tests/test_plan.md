# Checklist breve de pruebas y entrega

## 1) Checklist de pruebas mínimas
- [ ] El programa corre en terminal sin depender de internet.
- [ ] Existe al menos un caso de entrada válida documentado.
- [ ] Existe al menos un caso de entrada inválida documentado.
- [ ] El resultado coincide con lo esperado en casos básicos.

## 2) Checklist de documentación
- [ ] `docs/propuesta.md` está completo.
- [ ] `docs/caso_de_uso.md` está completo.
- [ ] `docs/plan_de_pruebas.md` incluye tabla con resultados.
- [ ] El alcance mínimo está claramente delimitado.

## 3) Checklist de ejecución
- [ ] `scripts/run.sh` existe y se ejecuta.
- [ ] `src/main.<ext>` corresponde al lenguaje elegido.
- [ ] El flujo principal del caso de uso se puede reproducir.

## 4) Checklist de entrega final
- [ ] Repositorio ordenado y sin archivos innecesarios.
- [ ] Sin frameworks, sin nube, sin APIs externas.
- [ ] Proyecto pequeño, viable y coherente con la propuesta.
- [ ] Commit final con documentación y prototipo mínimo (opcional).
